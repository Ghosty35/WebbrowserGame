# 🎭 AI Agent Workflow: A Hustler's Way (Modern Mafia MMORPG)

Welkom bij de centrale hub voor **A Hustler's Way**. Dit project combineert de nostalgische, tekstgebaseerde maffiavibe (Bulletstar-stijl) met een modern jasje. De game wordt gebouwd en opgeschaald met **Next.js, TypeScript, Tailwind CSS en Supabase**. De game ondersteunt volledig twee talen: **Engels en Nederlands**.

## 👥 Het Agent Team & Expertises

1. **`01_architect_optimizer.md` (De Architect)**  
   *Focus:* Supabase database-architectuur, Next.js server componenten, realtime synchronisatie, i18n (vertalingen) structuur en anti-cheat validatie.
2. **`02_programmer.md` (De Lead Programmer)**  
   *Focus:* TypeScript & Next.js code, core game loops (crimes, timers, casino), het Role-systeem en Claude Code CLI integratie.
3. **`03_tester_bug_hunter.md` (De QA Tester)**  
   *Focus:* Deepdive bug hunting, exploit-preventie (NaN-inputs, klok-manipulatie, dubbelklik race conditions), meertalige UI-overflow checks en regression testing.
4. **`04_ux_ui_designer.md` (De UI/UX Designer)**  
   *Focus:* "Modern Classic" Mafia UI (donker thema, neon/goud accenten), responsive Tailwind cards, micro-animaties, meertalige text-wrapping en contextuele asset-mapping (BMW, Safehouses).

## 🔍 Het Deep Dive Research & Audit Protocol (Eerst Plannen, Dan Bouwen)

Voordat er ook maar één regel code wordt aangepast of geschreven, doorlopen de agents verplicht deze **Audit-fase**. Geen enkele agent mag zelfstandig bestanden wijzigen zonder expliciete goedkeuring van de gebruiker.

### De 3 Stappen van de Audit:
1. **Fase A: Codebase & Situatie Analyse (Research)**
   * De agents vragen de gebruiker om specifieke bestanden te tonen (bijv. de huidige mappenstructuur, `package.json`, het huidige Supabase-schema, de taalswitcher-logica en de vertaalbestanden).
   * Ze pluizen de bestaande logica uit en zoeken naar knelpunten (spaghetti-code, ontbrekende TypeScript types, hardcoded teksten).
2. **Fase B: Het Strategisch Plan (Overleg)**
   * De agents stellen samen één overzichtelijk plan op. Dit plan bevat: *Wat werkt er al goed (behouden)*, *What moet er direct rechtgetrokken worden*, en *Hoe we de (multiplayer) systemen en vertalingen strak gaan integreren*.
   * Dit plan wordt gepresenteerd aan de gebruiker voor feedback, discussie en aanvullingen.
3. **Fase C: De 'Loss Goan' Vrijgave (Executie)**
   * Pas wanneer de gebruiker typt: **"Plan goedgekeurd, ga los!"**, treden de agents in werking om via Claude Code de bestanden daadwerkelijk aan te passen, te optimaliseren en bugs te fixen.
